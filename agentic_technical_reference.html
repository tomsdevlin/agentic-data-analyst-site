Agentic Pipeline — Technical Reference

Architecture, function signatures, data structures, and design decisions for the agentic/ module. For contributors and integrators.

System Design

A tool-calling agent loop for open-ended data analysis. The model receives a goal, dataset schema, and data dictionary; it then issues tool calls iteratively — running code, anchoring findings, and querying context — until it judges the investigation complete and calls synthesise. A separate charting phase and a post-synthesis reviewer complete the run.

The agentic pipeline is architecturally separate from the workflow pipeline (planner.py → executor_loop.py → synthesiser.py → reviewer.py). The agentic/ module imports shared infrastructure from agent.py (run_sandboxed, get_schema_with_quality) and config.py but never modifies them. All agentic-specific logic stays within agentic/.

Agent Loop Flow — Pseudocode

def run_agentic(config: AgentConfig) -> AgentResult:

  # 0. Resolve adapter and build system prompt
  adapter     = get_adapter(config.model_name)
  schema      = get_schema_with_quality(config.df, config.dataset_name)
  dict_text   = Path(config.dict_path).read_text() if config.dict_path else ""
  system_prompt = adapter.system_prompt(schema, config.goal, dict_text)

  # 1. Initialise shared state
  history     = TurnHistory(system_prompt)
  logger      = AgentLogger(run_id, config.goal, config.model_name)
  observations = []         # accumulated tool outputs for reviewer
  repeat_counter = {}       # snippet → count (repeat-loop guard)
  no_tool_count  = 0       # consecutive turns with no tool call
  synthesis   = None

  # 2. Analysis loop
  for turn in range(config.max_turns):
    response = client.chat(
      model=config.model_name,
      messages=history.build_messages(),
      tools=TOOL_SCHEMAS,
      options=adapter.options(),
      **adapter.extra_kwargs()            # think=False for qwen3; {} for gpt-oss
    )

    if not response.message.tool_calls:
      no_tool_count += 1
      if no_tool_count >= 2:
        history.add_nudge("URGENT: call a tool now.")
      history.add_assistant_text(response.message.content)
      continue
    no_tool_count = 0

    tc        = response.message.tool_calls[0]
    tool_name = tc.function.name
    tool_args = tc.function.arguments

    tool_result = dispatch_tool(tool_name, tool_args, config)

    # Strip figure_bytes before JSONL logging — PNG bytes not JSON-serialisable
    log_result  = {k: v for k, v in tool_result.items() if k != "figure_bytes"}
    logger.log_turn(turn, response, tool_name, tool_args, log_result)

    history.add_tool_turn(response.message, tool_name, tool_result)
    observations.append({"turn": turn, "tool": tool_name, "result": log_result})

    # Repeat-loop guard: 3 identical observations triggers a nudge
    snippet = str(tool_result)[:200]
    repeat_counter[snippet] = repeat_counter.get(snippet, 0) + 1
    if repeat_counter[snippet] >= 3:
      history.add_nudge("STOP retrying. Move on or call synthesise.")

    if tool_name == "synthesise":
      synthesis = tool_args["synthesis"]
      break

  # 3. Charting phase
  charts = generate_charts(synthesis, schema, config, adapter, logger)

  # 4. Model swap if reviewer_model != analyst model
  if config.reviewer_model and config.reviewer_model != config.model_name:
    _unload_model(config.model_name)

  # 5. Review + optional single revision round
  review = review_synthesis(synthesis, observations, config, logger, revision_round=0)

  if review.needs_revision:
    revised   = revise_synthesis(synthesis, review, observations, config)  # analyst model
    synthesis = revised
    review    = review_synthesis(revised, observations, config, logger, revision_round=1)

  logger.log_summary(synthesis, review, charts)
  return AgentResult(synthesis=synthesis, review=review, charts=charts, ...)

agent_loop.py Agent Loop

Entry point for all agentic runs. Contains AgentConfig, AgentResult, and run_agentic(). Imports from all other agentic/ sub-modules but contains no tool dispatch or review logic itself.

dict_path is the most important field after goal. If set, the plain-text data dictionary is read at run start and embedded in the system prompt via adapter.system_prompt(dict_text=...). Without it, the model must infer column semantics from names and types alone, which produces framing errors on ambiguous fields.

Model Swap — _unload_model()

When reviewer_model differs from model_name, the analyst model must be evicted from Ollama VRAM before the reviewer loads. On a 128GB machine running a 35B MoE model, failing to unload leaves insufficient headroom for the 120B reviewer, causing swap-induced hangs.

def _unload_model(model_name: str):
  # keep_alive=0 instructs Ollama to evict the model immediately after this call
  client.generate(model=model_name, prompt="", keep_alive=0)

# _active_model tracks which model is currently resident
# swap guard: only fires when reviewer_model != model_name
# revise_synthesis() always uses the analyst model — the reviewer judges, not rewrites

tools.py Tool Schemas & Dispatch

Defines TOOL_SCHEMAS (the JSON schema list passed to client.chat(tools=...)) and all _dispatch_*() functions. The dispatch router dispatch_tool(name, args, config) is the single call site in agent_loop.py.

Tool Schemas

Five tools in total. The model's understanding of available actions comes exclusively from these schema definitions — not from prompt rules.

tool_choice is not supported by the Ollama Python client. The OpenAI tool_choice: "required" parameter does not exist in ollama.Client.chat(). The required array in the schema prevents field omission but cannot prevent empty strings — that is why _dispatch_record_finding() validates and rejects at dispatch time, not at schema level.

Dispatch Functions

_dispatch_execute_code()

def _dispatch_execute_code(code: str, df: pd.DataFrame) -> dict:
  # Pre-inject scientific libraries into sandbox globals
  from scipy import stats
  import statsmodels.api as sm
  extra_globals = {"stats": stats, "sm": sm}

  # run_sandboxed() is imported from agent.py (isolation rule)
  result = run_sandboxed(code, df, extra_globals=extra_globals)

  # result["figure_bytes"] may be present — caller strips before log_turn()
  return result   # {ok, stdout, frames, figure_bytes?, error?}
scipy and statsmodels are pre-injected into every execute_code call. The model assumes both are in scope as stats and sm without importing. Without the pre-injection, OLS regressions and statistical tests produce NameError every time. This is agentic-specific behaviour kept in tools.py, not added to agent.py.

_dispatch_record_finding()

def _dispatch_record_finding(label: str, finding: str, store: list) -> dict:
  if not finding.strip():
    # Reject empty — enforce immediately with a directive, not a suggestion
    return {
      "error": (
        "finding cannot be empty. "
        "Immediately call record_finding again. "
        "Copy exact numbers from the immediately preceding execute_code output. "
        "Do not run more code before fixing this."
      ),
      "recorded_findings": store
    }

  store.append({"label": label, "finding": finding})
  return {"recorded_findings": store}   # full list returned each call

_dispatch_synthesise()

def _dispatch_synthesise(synthesis: str) -> dict:
  # Just returns the text — agent_loop handles the break
  return {"synthesis": synthesis, "status": "synthesis_complete"}

context.py TurnHistory

Manages the message list for each client.chat() call. The agent loop calls build_messages() every turn — the list grows as tool calls and results accumulate. No truncation is applied; context growth is monitored via tokens_in in turn logs.

TurnHistory(system_prompt: str)
MethodDescription
build_messages()Returns [{role:"system", content:system_prompt}, ...all turns]. Called each turn — the list is not modified in place.
add_tool_turn(assistant_msg, tool_name, tool_result)Appends the assistant message (with tool_calls) and the corresponding tool result message in Ollama format.
add_assistant_text(content)Appends a plain assistant message (no tool call). Used when the model responds with text instead of a tool — logged, counts toward no-tool guard.
add_nudge(message)Appends a user-role message injected by the loop guard. Used for repeat-loop nudges and no-tool-count urgents. Visible to the model as if from the user.

Tool Result Message Format

Ollama expects tool results as a message with role: "tool" and a name field matching the tool call. content is a JSON-serialised string of the result dict.

{
  "role":    "tool",
  "name":    "execute_code",                   # must match tool call name
  "content": "{\"ok\": true, \"stdout\": \"...\"}"  # JSON string
}
Context summarisation is not yet implemented. If tokens_in in the turn logs approaches AGENTIC_NUM_CTX = 32768, the model will degrade silently. Monitor logs before adding long-running goals.

logging.py AgentLogger

Writes a JSONL log to agentic_logs/. Each run produces one file; each turn, review, figure set, and summary is a separate record in that file distinguished by record_type. The HTML report generator (generate_agentic_report.py) reads this file to produce the per-goal cards.

Synthesis is sanitised before injection into the chart system prompt. _sanitise_synthesis() strips non-ASCII characters (e.g. ★, currency symbols) before passing the synthesis text to adapter.chart_system_prompt(). These characters crash Ollama's XML tool schema parser. The full synthesis (including non-ASCII) is used everywhere else — only the chart prompt injection is sanitised.
Schema is passed to the charting phase. Without it, the model guesses column names, fails on the first chart call, then spends all remaining turns listing columns to recover — producing zero charts. The schema is captured from the analysis phase and passed explicitly to generate_charts().

reviewer.py Reviewer & Revision

One LLM call per run (not per-finding). The reviewer receives the full synthesis and all accumulated observations, produces a JSON flag list, and the assessment is derived deterministically from the flag set — the model's stated overall_assessment field is ignored. One revision round is triggered if correctable flags with confidence ≥ 0.8 are present.

review_synthesis(
  synthesis: str,
  observations: list[dict],
  config: AgentConfig,
  logger: AgentLogger,
  revision_round: int = 0
) → AgenticReviewResult
revise_synthesis(
  synthesis: str,
  review: AgenticReviewResult,
  observations: list[dict],
  config: AgentConfig
) → str # revised synthesis text

revise_synthesis() always uses the analyst model (config.model_name), not the reviewer model. The analyst corrects its own work; the external reviewer then judges again. num_predict=8192 on all reviewer and revision calls — gpt-oss:20b runs thinking mode on non-tool prompts, exhausting 1024 tokens entirely on thinking with empty content.

Confidence Gate — _filter_flags()

The confidence gate is applied after the reviewer LLM call and before _derive_assessment(). It enforces two structural rules on correctable flags.

def _filter_flags(flags: list[dict]) -> list[dict]:
  filtered = []
  for flag in flags:
    if flag["type"] in CORRECTABLE_TYPES:

      # Rule 1: quote-triple required — drop if any field missing
      if not all(flag.get(k) for k in ["synthesis_quote", "evidence_quote", "conflict_reason"]):
        continue   # silently dropped

      # Rule 2: confidence gate — below threshold, downgrade to REVIEW (no revision)
      conf = flag.get("confidence", 0.0)
      if conf < _CONFIDENCE_THRESHOLD:   # _CONFIDENCE_THRESHOLD = 0.8
        flag["_review_only"] = True     # logged; does not trigger revision

    filtered.append(flag)
  return filtered

def _derive_assessment(flags: list[dict]) -> str:
  correctable_high = [f for f in flags
                      if f["type"] in CORRECTABLE_TYPES and not f.get("_review_only")]
  advisory = [f for f in flags if f["type"] in NON_CORRECTABLE_TYPES]
  low_conf = [f for f in flags if f.get("_review_only")]

  if   correctable_high: return "FAIL"     # triggers revision
  elif advisory or low_conf: return "REVIEW" # logged, no revision
  else: return "PASS"
Both the exception path and the parse-error path in review_synthesis() must use _derive_assessment([]), not a hardcoded "FAIL". An empty flag list → PASS. Hardcoding "FAIL" on reviewer call failure causes valid syntheses to be scored as failures when the reviewer itself breaks (e.g. JSON parse error on a legitimate synthesis).

JSON Sanitisation — _parse_review_json()

gpt-oss:20b writes \$295 instead of $295 inside JSON strings. \$ is not a valid JSON escape sequence; Ollama rejects it with HTTP 500 before Python can intercept it. The prompt rule ("Never escape currency symbols") is the primary fix; the sanitiser is defence-in-depth.

def _parse_review_json(json_str: str) -> dict:
  # Strip orphaned backslashes before any valid JSON escape sequence
  cleaned = re.sub(r'\\(?=[^"\\/bfnrtu])', '', json_str)
  return json.loads(cleaned)

def get_adapter(model_name: str) -> ModelAdapter:
  if "qwen3" in model_name.lower():
    return Qwen36Adapter()
  elif "gpt-oss" in model_name.lower():
    return GptOssAdapter()
  raise ValueError(f"No adapter for model: {model_name}")

Loop Guards

Repeat-loop guard (analysis)

After each tool call, the first 200 characters of the stringified result are compared against a counter. If the same snippet appears 3 times, a nudge is injected into the message history as a user-role message. Without this, qwen3.6 has been observed repeating the same truncated query 13 times on a monthly-trend goal.

No-tool guard

A dedicated _no_tool_count counter (separate from the repeat-loop counter) tracks consecutive turns where the model returns text without a tool call. After 2 consecutive no-tool turns, an URGENT nudge fires. The general repeat-loop guard's 3-turn window was too slow to stop this pattern.

Repeat-hash guard (charting)

Each code submission in the charting phase is normalised and hashed. If the hash was already seen, the loop breaks immediately — identical code cannot run again. A counter is also returned in the tool response so the model knows how many chart turns remain.

Sandbox Pre-injection

Every execute_code call in the agentic pipeline pre-injects two scientific libraries into the sandbox globals before the model's code runs. Both adapters document this in their system prompts.

Name in sandboxModuleWhy pre-injected
statsscipy.statsModel assumes stats is in scope for t-tests, Mann-Whitney, KS-tests. Without pre-injection: ~4 failed turns per run on complex goals.
smstatsmodels.apiOLS regression. Added 30 May 2026 after HR03 No module named 'statsmodels' failure. Model calls sm.OLS(...).fit() unprompted when the goal warrants regression.

Chart generation also pre-injects seaborn as _sns (private name, not advertised) to prevent import failures when model-generated chart code imports seaborn. The injection is in _dispatch_execute_code() inside charting.py.

AGENTIC_CHART_TURNS7Charting phase turn budget. Raised from 5 — 5 was too tight for exploratory chart generation.
AGENTIC_NUM_CTX32768Context window for agentic calls. Separate from OLLAMA_NUM_CTX (16384 for the workflow pipeline) — the agentic loop accumulates history across turns and needs more headroom.

Log Schema — JSONL Format

Turn record

{
  "record_type":               "turn",
  "run_id":                    str,
  "turn":                      int,
  "timestamp":                 str,    # ISO 8601
  "elapsed_s":                 float,  # wall-clock for this turn
  "cumulative_s":              float,
  "action":                    str,    # tool name called
  "tool_call":                 dict,   # {name, arguments}
  "result":                    dict,   # tool result (figure_bytes stripped)
  "tokens_in":                 int,    # prompt_eval_count or 0
  "tokens_out":                int,    # eval_count or 0
  "tokens_generated_cumulative":int,
  "ollama_eval_duration_ms":   int|null,
  "ollama_load_duration_ms":   int|null,  # non-zero = model was loaded/swapped this turn
  "error":                     str|null
}

Review record

{
  "record_type":        "review",
  "overall_assessment": str,          # "PASS" | "REVIEW" | "FAIL"
  "flags":              list[dict],    # filtered flag objects
  "reviewer_note":      str,
  "reviewer_model":     str,
  "elapsed_s":          float,
  "revision_round":     int            # 0 = initial review, 1 = post-revision re-review
}

Summary record

{
  "record_type":        "summary",
  "goal":               str,
  "dataset_name":       str,
  "model":              str,
  "reviewer_model":     str,
  "synthesis":          str,          # final synthesis text
  "overall_assessment": str,
  "revision_count":     int,          # 0 or 1
  "citation_summary":   dict,         # CitationCheck.details
  "turns_used":         int,
  "total_time_s":       float,
  "timestamp":          str
}

Figures record

{
  "record_type": "figures",
  "figures":     [str, ...]   # base64-encoded PNG strings; read back by generate_agentic_report.py
}

Entry Points & CLI

ScriptPurposeKey flags
run_agentic_pipeline.py CLI runner for the agentic pipeline. Runs one or all 12 CORGI goals. Default model: AGENTIC_DEFAULT_MODEL. Populates dict_path from DICT_FILES mapping for CORGI datasets. --id FD01 · --model qwen3.6:35b-mlx · --reviewer-model gpt-oss:120b · --max-turns 30 · --verbose
generate_agentic_report.py Reads agentic JSONL logs and produces an HTML report with per-goal cards, inline charts (from base64 figures record), review flags, and turn traces. --log agentic_logs/foo.jsonl · --out report.html
test_reviewer.py 17-case calibration harness for the agentic reviewer. Covers 7 SHOULD_PASS false-positive guards, 4 SHOULD_FAIL true-positive checks, 3 clean synthesis, and 3 edge cases from observed false-positive patterns. --model gpt-oss:120b · --model qwen3.6:35b-mlx
app.py (Tab 3) Streamlit UI for the agentic pipeline. Dataset selector, model selector, goal input, and inline report rendering. Shares the sandbox and vector stores with Tabs 1 and 2. streamlit run app.py

Example runs

# Single goal, self-review
python run_agentic_pipeline.py --id FD01 --verbose

# Cross-review: qwen3.6 analyst, gpt-oss:120b reviewer
python run_agentic_pipeline.py --id HR03 \
  --model qwen3.6:35b-mlx \
  --reviewer-model gpt-oss:120b

# 4-goal dev subset (FD01, FD03, HR03, CL03) — standard regression batch
python run_agentic_pipeline.py --ids FD01 FD03 HR03 CL03

# All 12 CORGI goals overnight
caffeinate -i python run_agentic_pipeline.py

# Reviewer calibration test
python test_reviewer.py --model gpt-oss:120b

# Generate HTML report from latest log
python generate_agentic_report.py --latest
